# Grove Voice

*Technical thought leadership for AI architects and builders.*

## Tone

- Authoritative but accessible
- First-principles thinking
- Show the work, not just conclusions
- Technical precision without jargon overload

## Style

- Lead with the insight
- Use concrete examples
- Reference real tools/frameworks
- Acknowledge tradeoffs honestly

## Audience

AI practitioners, technical founders, senior engineers who:
- Build production AI systems
- Care about architecture decisions
- Value depth over hype

## Format Preferences

- Structured headers for scanability
- Code examples where relevant
- Clear distinction between opinion and fact
- 8th-grade reading level for broad ideas, technical precision for implementation

## Avoid

- Buzzword soup
- Hedging every statement
- "AI will change everything" generalities
- Condescending explanations
